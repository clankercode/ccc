ccc — call coding CLIs

One command to run any coding CLI. 20 language implementations.

Usage

ccc [--show-thinking] [runner] [+thinking] [:provider:model] [@alias] "<Prompt>"
ccc --help
ccc -h

At its simplest:

ccc "Fix the failing tests"

Slots

Arguments are positional slots parsed in order before the prompt:

SlotDescription
runnerSelect which coding CLI to use (default: oc)
opencode (oc), claude (cc), kimi (k), codex (c/cx), roocode (rc), crush (cr)
+thinkingSet thinking level: +0 (off) through +4 (max)
--show-thinkingRequest visible thinking output when supported; default off
:provider:modelOverride provider and model
@aliasUse a named preset from config

Examples

# Default runner (opencode)
ccc "Fix the failing tests"

# Explicit runner
ccc oc "Refactor auth module"

# Claude with thinking level and model override
ccc cc +2 :anthropic:claude-sonnet-4-20250514 "Add tests"

# Visible thinking on supported implementations
ccc --show-thinking "Fix the failing tests"

# Kimi with max thinking
ccc k +4 "Debug the parser"

# Print the canonical example config
ccc --print-config

# Codex
ccc codex "Write a unit test"

Config

Python and Rust load configuration from a project-local .ccc.toml searched upward from the current directory, then XDG_CONFIG_HOME/ccc/config.toml, then ~/.config/ccc/config.toml. Run ccc --print-config to print the canonical example config with the current supported keys.

Human-formatted output honors FORCE_COLOR and NO_COLOR before falling back to TTY detection; raw output modes are unchanged.

SettingDescription
Default runnerWhich CLI to use when no runner is specified
AliasesNamed presets referenced with @alias
Custom abbreviationsUser-defined runner shorthand names
Default provider/modelFallback provider and model selection
Visible thinkingshow_thinking defaults to off and enables runner-specific visible-thinking flags when supported
Canonical sections[defaults], [abbreviations], and [aliases.<name>]

Building & Running

Each language directory contains its own build tooling. Pick whichever language you prefer:

# Python
PYTHONPATH=python python3 -m call_coding_clis.cli "Your prompt"

# Rust
cd rust && cargo run --bin ccc -- "Your prompt"

# Go
cd go && go run ./cmd/ccc "Your prompt"

# TypeScript
node typescript/src/ccc.js "Your prompt"

# C
cd c && make && ./ccc "Your prompt"

# C++
cmake -B cpp/build -S cpp && cmake --build cpp/build && ./cpp/build/ccc "Your prompt"
