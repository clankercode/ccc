## 2. Library API

### Types (`src/CallCodingClis/Types.purs`)

```purescript
type CommandSpec =
  { argv :: Array String
  , stdinText :: Maybe String
  , cwd :: Maybe String
  , env :: StrMap String
  }

type CompletedRun =
  { argv :: Array String
  , exitCode :: Int
  , stdout :: String
  , stderr :: String
  }

type StreamCallback = String -> String -> Effect Unit
```

No newtype wrappers needed — these are simple records matching the other implementations.

### Runner (`src/CallCodingClis/Runner.purs`)

```purescript
newtype Runner = Runner
  { runFn :: CommandSpec -> Aff CompletedRun
  , streamFn :: CommandSpec -> StreamCallback -> Aff CompletedRun
  }
```

`run` and `stream` are async because Node's `child_process.spawn` is event-driven. `run` collects all output then resolves; `stream` fires the callback per chunk then resolves.

### buildPromptSpec (`src/CallCodingClis/PromptSpec.purs`)

```purescript
buildPromptSpec :: String -> Either String CommandSpec
buildPromptSpec prompt =
  let trimmed = String.trim prompt
  in if String.null trimmed || String.trim trimmed == ""
     then Left "prompt must not be empty"
     else Right { argv: ["opencode", "run", trimmed]
                , stdinText: Nothing
                , cwd: Nothing
                , env: empty }
```

Returns `Either String CommandSpec` — `Left` on empty/whitespace, `Right` on success. Mirrors Rust's `Result<_, &str>`.

## 3. FFI for Node.js child_process

### Approach: Thin FFI shim (Plan B)

Since PureScript's standard package sets rarely carry `node-child-process`, write a minimal FFI module.

**`src/FFI/Spawn.purs`**:

```purescript
foreign import spawn
  :: String                     -- command
  -> Array String               -- args
  -> SpawnOptions
  -> (SpawnEvent -> Effect Unit) -- onEvent callback
  -> Effect Unit                -- returns void; events arrive via callback

foreign import data SpawnEvent :: Type

foreign import eventStdout :: SpawnEvent -> String
foreign import eventStderr :: SpawnEvent -> String
foreign import eventClose  :: SpawnEvent -> Int
foreign import eventError  :: SpawnEvent -> String
foreign import isStdout    :: SpawnEvent -> Boolean
foreign import isStderr    :: SpawnEvent -> Boolean
foreign import isClose     :: SpawnEvent -> Boolean
foreign import isError     :: SpawnEvent -> Boolean

type SpawnOptions =
  { cwd :: Maybe String
  , env :: Maybe (StrMap String)
  , stdinText :: Maybe String
  }
```

**`src/FFI/Spawn.js`**:

```javascript
exports.spawn = function (command) {
  return function (args) {
    return function (opts) {
      return function (onEvent) {
        return function () {
          const child = require('child_process').spawn(command, args, {
            cwd: opts.cwd,
            env: opts.env
              ? { ...process.env, ...opts.env }
              : process.env,
            stdio: 'pipe'
          });

          child.stdout.on('data', (chunk) => {
            onEvent({ tag: 'Stdout', text: chunk.toString() })();
          });
          child.stderr.on('data', (chunk) => {
            onEvent({ tag: 'Stderr', text: chunk.toString() })();
          });
          child.on('close', (code) => {
            onEvent({ tag: 'Close', exitCode: code ?? 1 })();
          });
          child.on('error', (err) => {
            onEvent({ tag: 'Error', message: err.message })();
          });

          if (opts.stdinText) {
            child.stdin.write(opts.stdinText);
          }
          child.stdin.end();
        };
      };
    };
  };
};

exports.isStdout = function (e) { return e.tag === 'Stdout'; };
exports.isStderr = function (e) { return e.tag === 'Stderr'; };
exports.isClose  = function (e) { return e.tag === 'Close'; };
exports.isError  = function (e) { return e.tag === 'Error'; };
exports.eventStdout = function (e) { return e.text; };
exports.eventStderr = function (e) { return e.text; };
exports.eventClose  = function (e) { return e.exitCode; };
exports.eventError  = function (e) { return e.message; };
```

The FFI returns `Effect Unit` and communicates results via callbacks. The PureScript side wraps this into `Aff CompletedRun` using `Aff.makeAff`.

### Aff wrapper pattern

```purescript
runSpawn :: String -> Array String -> SpawnOptions -> Aff CompletedRun
runSpawn cmd args opts = makeAff \cb -> do
  let handleEvent ev
        | isStdout ev = pure unit  -- accumulate in local ref
        | isStderr ev = pure unit
        | isClose ev  = cb $ Right { argv: ..., exitCode: eventClose ev, ... }
        | isError ev  = do
            let msg = "failed to start " <> cmd <> ": " <> eventError ev <> "\n"
            cb $ Right { argv: [cmd] <> args, exitCode: 1, stdout: "", stderr: msg }
        | otherwise = pure unit
  spawn cmd args opts handleEvent
  pure mempty
```

Accumulate stdout/stderr using `Ref` (Effect) inside the Aff.

## 4. ccc CLI as a PureScript-compiled Node.js Entrypoint

**`src/CCC.purs`**:

```purescript
module CCC where

import Prelude
import Effect (Effect)
import Effect.Console (error)
import Effect.Uncurried (runEffectFn1)
import Node.Process (argv, exit)
import CallCodingClis.PromptSpec (buildPromptSpec)
import CallCodingClis.Runner (Runner, defaultRunner)

main :: Effect Unit
main = do
  args <- map (drop 2) argv  -- drop "node" and script path
  case args of
    [prompt] ->
      case buildPromptSpec prompt of
        Left err -> do
          error err
          exit 1
        Right spec -> do
          run <- defaultRunner
          result <- run spec  -- Aff via launchAff_
          -- forward stdout/stderr, exit with exitCode
          exit result.exitCode
    _ -> do
      error "usage: ccc \"<Prompt>\""
      exit 1
```

**spago build → output/Node/CCC/index.js** is the entrypoint. A wrapper shell script or package.json `bin` field aliases it as `ccc`.

The CLI uses `process.exit()` (via `Node.Process.exit`) to forward the child's exit code, matching all other implementations. Stdout/stderr forwarding happens inside the `stream` path: each chunk callback writes directly to `process.stdout` / `process.stderr`.

## 5. Prompt Trimming and Empty Rejection

- Use `Data.String.trim` from the `strings` package.
- After trimming, check `String.null (String.trim prompt)` or `String.trim prompt == ""`.
- PureScript `String.trim` handles all Unicode whitespace, matching the other implementations (Python's `.strip()`, Rust's `.trim()`, etc.).

## 6. Error Format: argv[0] Only

The error string for spawn failure must be exactly:
```
failed to start <argv[0]>: <error_message>\n
```

Where `<argv[0]>` is just the command name (first element of the argv array), not the full argv. This matches all other implementations. The trailing newline is required for consistency with the contract tests.

## 7. Exit Code Forwarding

The CLI must call `process.exit(result.exitCode)` to forward the child process exit code. This is critical because:
- If the child exits with code 42, `ccc` must exit with code 42
- `process.exit()` is the only reliable way in Node.js; `return` from main doesn't work since the event loop may not drain
- PureScript's `Node.Process.exit :: Int -> Effect Unit` maps directly to this

## 9. PureScript-Specific Considerations

### Effect system

- All FFI calls live in `Effect`. Async subprocess work is lifted to `Aff` via `makeAff` / `launchAff_`.
- No need for `Aff` in the pure `buildPromptSpec` function — it returns `Either`.

### FFI safety

- The spawn FFI uses uncurried function application (`runFn4`, etc.) to match JS arity exactly.
- The FFI callback is invoked from Node's event loop, so it must be called via `runEffectFn1` to preserve the PureScript continuation correctly.
- `mkAff` callback must invoke `cb` exactly once. The FFI guarantees this because Node fires exactly one `close` or `error` event.

### String handling

- PureScript `String` is UTF-16 in JS runtime. Buffer chunks from `child_process` arrive as `Buffer` and are converted via `.toString()` in the FFI layer, matching the TypeScript implementation.

### Build toolchain

- `spago build` compiles to `output/` directory
- `spago test` runs unit tests via Node.js
- The CLI entrypoint is `node purescript/output/Node/CCC/index.js`
- No bundler needed; PureScript compiles to CommonJS modules

### StreamCallback in Aff context

The `stream` function needs to call a `StreamCallback` (which is `Effect Unit`) from within an `Aff`. This is straightforward: use `liftEffect` to call the callback when data arrives.

## 10. Parity Gaps to Watch For

### Must have (contract requirements)

| Gap | Risk | Mitigation |
|-----|------|------------|
| Error format exact match | High — contract test checks for `"failed to start"` substring | Mirror the C/TS error string format exactly in FFI layer |
| Exit code forwarding | High — `process.exit()` is required, not `return` | Use `Node.Process.exit` |
| Trailing newline on errors | Medium — C and Python include `\n` | Explicitly append `"\n"` in error messages |
| `CCC_REAL_OPENCODE` env override | High — contract tests depend on it | Read env var before building argv |

### Nice to have (feature parity)

| Gap | Notes |
|-----|-------|
| `Runner.stream` true streaming | Rust's "stream" is non-streaming; C has no stream. Minimum viable: collect-then-callback (matching Rust). Stretch goal: per-chunk callbacks via FFI events. |
| `stdinText` support | Supported in all other implementations. The FFI must write to `child.stdin` before closing it. |
| `cwd` / `env` overrides | Straightforward via `spawn` options. |
| `CCC_RUNNER_PREFIX_JSON` | TypeScript supports this; other implementations don't. Omit for v1. |

### Known PureScript pitfalls

| Pitfall | Mitigation |
|---------|------------|
| FFI type mismatches causing runtime crashes | Use `Foreign`/`Data.Foreign` for validation if needed; keep FFI surface minimal |
| `Aff` not completing because callback never fires | Ensure the FFI always fires `close` or `error` (Node guarantees this for spawn) |
| Spago package set version drift | Pin a specific package set in `packages.dhall` |
| `process.exit()` killing pending writes | Flush stdout/stderr before calling exit (use `launchAff_` and await stream completion) |
